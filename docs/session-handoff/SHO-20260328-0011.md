- `approve` 后重新入队、进入新的 queued execution host 是预期行为。新的 host 会重新构造 orchestrator 和 memory bundle，因此旧 worker 内的短期 WM 本来就不可见。
- 当前真正的架构偏差是：控制层只持久化了 `decision/session status`，没有持久化完整 continuation authority；恢复路径却把短期 WM 里的 `workflow.task_specs` 当成 authoritative checkpoint 读取。
- 当前多代理收敛结论：`resume` 应走 `control-plane snapshot -> artifact refs -> hydrate WM -> continue`，而不是 `persist WM -> use WM as checkpoint`。

# Subtask Snapshots
## 1. Root Cause定位
- 最新错误发生在 `approve` 后的新 worker run 中，报错为 `Missing persisted workflow.task_specs for resume action: approve`。
- 关键日志位置在 `backend/logs/mas/mas_workflow.log`：
  - `waiting_gate` 停住
  - 用户 `POST /runtime/script/decision`
  - task 重新入队
  - 新 host 启动
  - orchestrator 读取 `workflow.task_specs` 失败
- 代码对应：
  - 读取点：`backend/app/agents/orchestrator.py`
  - 写入点：`backend/app/services/orchestration_state_adapter.py`
  - 短期内存实现：`backend/app/services/memory_provider.py` 和 `backend/app/agents/memory/storage/in_memory/`

## 2. Boundary结论
- `queue/worker` 只负责 requeue、eligibility、host bootstrap，不拥有 `approve/revise/replan` 语义。
- `control-plane` 才是 `checkpoint/resume authority` owner。
- `artifact/published deliverable` 是内容事实源，例如 approved script、payload ref、review contract ref。
- `WM` 只是执行期工作区，可以被重建，不应成为跨 worker 恢复的 authority。

## 3. 当前偏差
- 当前实现已经有 artifact/deliverable 机制，并且 `approve/revise/replan` 也会把部分 ref/contract 写进 `runtime_session.input_payload`。
- 但 continuation 所需的编排语义仍依赖 `workflow.task_specs`，而它只存在于短期 MAS WM。
- 这导致控制层持久化只做了一半：内容 ref 已落地，恢复语义还没落地。

## 4. 下一步设计目标
- 不要“把 MAS WM 持久化”当目标。
- 要补的是一份最小 `control-plane continuation snapshot`，并让恢复统一走：
  - load control-plane snapshot
  - resolve published artifact refs
  - hydrate fresh MAS WM
  - continue orchestration

## 5. 建议的最小 snapshot 字段方向
- `resume_action`
- `resume_from_node`
- `invalidation_scope`
- `target_agents` 或等价 continuation path
- `published_deliverable_refs`
- `review_contract`
- 少量无法稳定重算的 `runtime_hints`

# Avoid Repeat
- 不要把 `snapshot` 做成第二套业务模型；它只能是控制层的最小 continuation projection。
- 不要把整份 MAS WM 持久化来“修恢复”；WM 只能被重建，不能升级成 checkpoint SoT。
- 不要让 artifact 或 queue 反向承接 continuation 语义；artifact 只承载内容事实，queue 只做 transport/container。
