## 3. 已确认的设计偏差
- `session.input_payload.published_deliverables["script"]` 当前在 gate-open 时就会写入 candidate ref，approve 时再覆盖成 approved ref。

## 4. `approve / revise / replan` 语义收敛
- `approve`：允许当前节点产物成为 downstream input。
- `revise`：带反馈重跑当前节点 producer。
- `replan`：带反馈重跑更上游 planning + 当前节点 producer。
- 它们都不是“直接改记忆或资产”的语义。

## 5. `035` 已冻结的最小方案方向
- control-plane write path 收紧 `session.input_payload.published_deliverables`：
  - 只投影 approved downstream refs
  - 不再在 gate-open 时把 candidate script ref 投影到 runtime input
- review 期 candidate 继续停留在现有 surfaces：
  - gate `artifact_refs`
  - attempt outputs
  - gate facts / preview surfaces
- runtime/orchestration 必须守住 mandatory producer prerequisite：
  - script-consuming nodes 不能在 `script` producer 未完成且未形成 downstream-eligible input 前进入执行
