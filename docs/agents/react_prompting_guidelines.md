ReAct 提示与反思约定（简版）

目标

- 让子代理以 ReAct 循环（Observe → Think → Plan → Act → Reflect）自主完成任务；
- 提示中仅包含“目标、事实、约束与进度/反思摘要”，不出现工具名、参数名或取值范围；
- 工具能力与校验全部通过 tools schema 注入与工具层校验，不在提示中描述。

提示形态

- 内容：
  - 任务目标：要达成的可测完成条件（如“所有场景均生成成功或明确失败”）。
  - 事实上下文：场景编号、依赖/连续性信号、简要描述；不含能力/参数信息。
  - 进度与反思：上一轮完成/失败统计、最后几条“反思摘要”。
  - 迭代语义：允许分轮执行与部分覆盖；未完成场景留待后续轮次。
- 不包含：
  - 工具名、函数名、参数名与取值范围；
  - 顺序化流程指令（例如“先A再B”）。

反思与记忆

- 每轮 REFLECT 产出：
  - 本轮是否有进展（成功个数）；
  - 剩余待处理数量；
  - 简短“反思摘要”（如“未取得进展，建议先处理无依赖场景”）。
- 写回 working_state：
  - reflection_notes（保留最近若干条）；
  - no_progress_rounds（连续无进展轮次，用于提前停止）。

终止条件

- 剩余待处理为空且无新依赖待解锁；或连续无进展达到阈值（默认2轮）触发提前停止，返回部分结果。

执行层约定

- 不改变 LLM 自主规划；执行层仅做“结果对齐”（例如同轮内先出现连续性准备、再出现视频生成时，将前者产出的 image_url 注入后者同一场景的参数）。
- 所有约束（如时长枚举）由工具 schema 与工具层校验，不在提示中表达。

适用范围

- 适用于本项目 Image/Video 子代理及其他需要 ReAct 的 Agent；
- 若需新增 Agent，请遵循本规范并通过 tools schema 注册能力，避免在提示中“硬编码能力与参数”。
