## 1. Current Problem
- Symptom:
  - `quality_checker` can still emit `Missing introduction scene` and `Missing conclusion scene` for a completed workflow even when the scene sequence is present and durations are correct.
  - The reproduced real sample is workflow/task `9587c5b5-216c-4bf4-8a37-4c9078b4e900`.
- Impact:
  - QC reports narrative-semantic defects that are not proven by the available contract.
  - These issues are treated as normal QC findings and can trigger misleading recommendations / human-review escalation even though the underlying input is only unlabeled.
- Reproduction:
  - Real sample artifacts:
    - `backend/storage/temp/published_deliverables/script_9587c5b5-216c-4bf4-8a37-4c9078b4e900_attempt106_rev0.json`
    - `backend/storage/temp/context/video_composer_scene_media_9587c5b5-216c-4bf4-8a37-4c9078b4e900.json`
  - Current characterization tests freeze that:
    - `concept_plan.scenes[*].scene_type` is absent
    - `scene_overview.scenes[*].scene_type` is absent
    - `composition_timeline[*].scene_type` is absent after QC boundary assembly
    - QC still flags intro/outro as missing because unlabeled entries are defaulted to `main_content`

## 2. Deviation Analysis
- Expected behavior:
  - QC should only report missing intro/outro when the available contract provides enough explicit `scene_type` evidence to support that conclusion.
  - If scene labels are absent, QC should not silently convert “unlabeled” into “semantic failure”.
- Actual behavior:
  - `build_quality_checker_context(...)` rebuilds `composition_timeline` and only carries `scene_type` when upstream records already have it.
  - `_analyze_content_quality(...)` then evaluates `scene_types = entry.get("scene_type", "main_content")`, which turns every unlabeled scene into `main_content`.
  - `_analyze_scene_breakdown(...)` uses the same default, so unlabeled timelines look like an all-`main_content` composition rather than an unknown-label composition.
- Root cause hypothesis:
  - Primary fault path:
    - `backend/app/agents/quality_checker.py::_analyze_content_quality`
    - `backend/app/agents/quality_checker.py::_analyze_scene_breakdown`
    - owner defect is the heuristic that equates missing labels with missing intro/outro semantics
  - Possible upstream dependency:
    - The current script-stage / scene-overview contract does not reliably provide `scene_type`, even though `concept_planner` supports it when present.
    - This dependency is frozen as a possible future follow-on, not the first owner to repair under `057`.

## 3. Implementation Approach
- Patch strategy:
  - Stage A is already complete before implementation:
    - RCA freeze and owner verdict are recorded in the session and companion tests.
  - Stage B stays inside the QC semantic boundary by default:
    - change QC scene-type handling so unlabeled timelines are treated as “unknown / unverified label coverage”, not as implicit `main_content`
    - preserve explicit intro/outro checks when labels are actually present
    - add bounded diagnostics so missing labels remain visible without being misreported as semantic defects
  - Do not change MAS WM ownership, published deliverable ownership, or runtime/control-plane behavior in this plan unless Stage B proves a wider owner is unavoidable.
- Compatibility considerations:
  - Do not reintroduce direct reads from published/temp artifacts into agent live paths.
  - Do not add case-specific hardcodes for workflow `9587...` or this trailer shape.
  - Preserve existing technical-quality diagnostics and existing labeled-scene behavior.
  - Other agents continue to consume context from MAS WM boundaries only.
- Rollback plan:
  - If the bounded QC repair cannot solve the false-positive class without changing upstream contracts, stop Stage B and open a narrower contract-stabilization successor instead of widening `057`.
