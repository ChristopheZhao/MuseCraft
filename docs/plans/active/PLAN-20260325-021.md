  - [backend/app/agents/orchestrator.py](/mnt/d/code/agent/Opensource/vertical_application/short-video-maker/backend/app/agents/orchestrator.py) 仍可能把 `runtime_overrides` 注回 agent input

## 2. 架构边界与原则
- 本计划服从：
  - [single_episode_harness_architecture_20260311.md](/mnt/d/code/agent/Opensource/vertical_application/short-video-maker/docs/architecture/single_episode_harness_architecture_20260311.md)

- Canonical contract boundary:
  - generic execution hint 只允许存在于 planning/control-plane/assembler 上游输入，正式载体是 `runtime_hints`
  - leaf-agent 真正消费的唯一控制面对象仍是 `ExecutionContract`
  - `project` 只能是 wrapper；不得通过松散 dict 重写 per-episode live contract
  - `ContextContractAssembler` 仍是 supporting capability，不是新层，也不应继续承担长期 compat bridge
- Hard rules:
  - 不新增新的 compat adapter / bridge
  - 不允许把 `runtime_hints` 误落成新的 leaf-facing hints bag
  - 不允许把 typed project options 重新穿透成 wrapper-owned live mini-contract
  - 若必须兼容，`runtime_overrides` 最多只能作为 entry-boundary 一次性、显式、短期 translator 输入，不得在内部继续传播

## 4. 当前残余分类
### 4.1 Project entry open bag
- 文件：
  - `backend/app/api/v1/endpoints/projects.py`
  - `src/types/project.ts`
- 问题性质：
  - request contract 仍以 `Record<string, any>` / `Dict[str, Any]` 暴露 `runtime_overrides`
  - project wrapper 入口还在 author 一个无类型的 execution/option 混合包

### 4.2 Wrapper passthrough / scatter injection
- 文件：
  - `backend/app/agents/episode_orchestrator.py`
- 问题性质：
  - wrapper 将 legacy bag 透传到 single-episode payload
  - `payload.update(runtime_overrides)` 让 wrapper 直接改写 leaf-facing input surface

### 4.3 Generic boundary fallback
- 文件：
  - `backend/app/services/context_assembler.py`
  - `backend/app/agents/orchestrator.py`
- 问题性质：
  - assembler 仍 dual-read `runtime_hints` / `runtime_overrides`
  - orchestrator 在没有 execution contract 时仍可能把 `runtime_overrides` 注回 agent input
  - 这会把 entry compat surface 延长成内部隐式协议

### 4.4 Mixed test/contract anchors
- 文件：
  - `backend/tests/unit/test_video_generator_audio_route_injection.py`
  - `backend/tests/unit/test_video_composer_execution_boundary.py`
  - 其他直接以 `runtime_overrides` 构造 contract 的测试
- 问题性质：
  - 当前测试里仍把 `runtime_overrides` 当作 canonical contract 输入
  - 迁移后这些测试应转成 migration coverage，而不是 truth anchor

## 5. 执行计划
### Phase A: 冻结 vocabulary 与 ownership
- 明确冻结：
  - `runtime_hints` = generic execution-hint carrier
  - `ExecutionContract` = leaf-agent 唯一控制面对象
  - typed project options = wrapper/service 边界专属选项
  - `runtime_overrides` = legacy ingress only，不再是内部 contract
- 验收：
  - 不再允许把 `runtime_overrides` 描述成 generic live contract

### Phase B: 收紧 project entry contract
- 将 project orchestrate request 从 `runtime_overrides` open bag 收敛成：
  - 显式 typed project options / fields
  - 若必须兼容，入口处做一次性 translator，并记录 diagnostics / deprecation
- 验收：
  - frontend/backend 类型不再把 project request 暴露为任意 `runtime_overrides` bag

### Phase C: 删除 wrapper passthrough 与散弹注入
- 删除 episode wrapper 中的：
  - `runtime_overrides` 整包透传
  - `payload.update(runtime_overrides)` 这类散弹式注入
- 只保留白名单映射：
  - project-only option 留在 project wrapper/service
  - generic execution hint 进入 `runtime_hints`
- 验收：
  - project wrapper 不再直接 author leaf-facing ad hoc keys

### Phase D: 收口 generic boundary 到 runtime_hints / ExecutionContract
- 将 `ContextContractAssembler` 收到：
  - generic hint 只读 `runtime_hints`
  - `build_execution_contract(...)` 只接受 canonical hint 输入
- 删除 orchestrator 对 `runtime_overrides` 的 fallback reinjection
- 验收：
  - 内部 canonical path 变成：
    - entry typed option / translator
    - `runtime_hints`
    - `ExecutionContract`
    - leaf-agent

### Phase E: 测试迁移与兼容退出
- 将旧测试从“`runtime_overrides` 是真值输入”改成：
  - canonical `runtime_hints` contract 测试
  - entry-boundary compatibility translator 测试（若 translator 仍存在）
- 刷新 repo grep / focused validation，确认 `runtime_overrides` 不再出现在内部 active path

## 6. 验收标准
- generic execution hint 只通过 `runtime_hints` 进入 canonical path。
- leaf-agent 不再读取、也不再接收到 `runtime_overrides`。
- `ContextContractAssembler` 不再 dual-read `runtime_hints` / `runtime_overrides`。
- orchestrator 不再把 `runtime_overrides` 注回 agent input。
- project wrapper 不再通过 `payload.update(...)` 之类方式改写 single-episode leaf-facing input。
- 若兼容 translator 仍存在，它只位于 entry boundary，带显式 diagnostics，并有退出条件。
- `runtime_overrides` 不被并入 `020`，且不被错误降格为纯 project host 问题。

## 7. 风险与护栏
- 风险 1：
  - 把 `runtime_hints` 误变成新的 leaf-facing hints bag。
  - 护栏：leaf side 只认 `ExecutionContract`。
- 风险 2：
  - 把 typed project option 再次透传成 wrapper-owned live contract。
  - 护栏：typed option 只属于 wrapper/service 边界。
- 风险 3：
  - 用 endpoint translator + assembler fallback + orchestrator reinjection 形成新的 multi-hop compat bridge。
  - 护栏：compat 只允许 entry-boundary 单点、短期、显式 translator。
- 风险 4：
  - 误把本计划并入 `020`，导致 host cleanup 与 contract cleanup 混在一起。
  - 护栏：本计划是独立 successor；`020` 继续只做 residual host / island cleanup。
