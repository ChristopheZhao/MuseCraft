## 1. Current Problem
- Symptom:
  - 本地 `npm run dev` 首次编译 `/console` 时，Next 会多次打印 `The user aborted a request` / `Failed to download Inter from Google Fonts`，随后退回 fallback font。
  - quick-mode 表单加载时，控制台会打印 `Skipped ".pdf"`, `Skipped ".doc"`, `Skipped ".docx"` because it is not a valid MIME type。
- Impact:
  - Google Fonts 外网依赖会拖慢 dev 首次编译，且在受限网络环境下制造误导性“用户中止请求”日志。
  - 上传区的文档类型过滤当前配置无效，`.pdf/.doc/.docx` 没有被正确注册到 `react-dropzone` 的 accept schema。
  - 两个问题都属于前端基线 hygiene，不应继续污染已关闭的 [PLAN-20260406-058.md](/mnt/d/code/agent/opensource/vertical_application/short-video-maker/docs/plans/active/PLAN-20260406-058.md) 或 canvas discovery 的 [PLAN-20260406-059.md](/mnt/d/code/agent/opensource/vertical_application/short-video-maker/docs/plans/active/PLAN-20260406-059.md)。
- Reproduction:
  - 启动 `npm run dev`，首次访问 `/console`，观察 Next dev server 输出中的 Google Fonts fetch abort / fallback 日志。
  - 打开 quick-mode 表单，观察上传控件初始化时打印的 `.pdf/.doc/.docx` MIME warning。

## 2. Deviation Analysis
- Expected behavior:
  - 本地 dev 启动不应依赖 Google Fonts 外网拉取；即便离线或受限网络，也不应在首屏编译阶段反复重试远程字体下载。
  - 上传控件应以 `react-dropzone` 支持的 MIME-to-extension 结构声明文档类型，而不是把扩展名当成 MIME type。
- Actual behavior:
  - [layout.tsx](/mnt/d/code/agent/opensource/vertical_application/short-video-maker/src/app/layout.tsx) 使用 `next/font/google` 的 `Inter`，导致 `/console` 首次编译时触发 Google Fonts 请求。
  - [VideoRequestForm.tsx](/mnt/d/code/agent/opensource/vertical_application/short-video-maker/src/components/forms/VideoRequestForm.tsx) 传入 `['image/*', 'video/*', '.pdf', '.doc', '.docx']`，而 [FileUploadZone.tsx](/mnt/d/code/agent/opensource/vertical_application/short-video-maker/src/components/ui/FileUploadZone.tsx) 又把这些值直接折叠成 `accept` 对象 key，导致扩展名被错误当成 MIME type。
- Root cause hypothesis:
  - 问题 1 primary owner：
    - 前端布局层选择了远程字体提供方式，不适配当前本地开发与受限网络环境。
  - 问题 2 primary owner：
    - 上传组件的 accept contract 设计过于宽泛，只接受字符串数组，没有区分 MIME 和扩展名语义。

## 3. Implementation Approach
- Patch strategy:
  - 将 app layout 的字体策略改为不依赖 Google Fonts 远程下载的本地/系统方案。
  - 收敛上传 accept contract，改成明确的 MIME + extension 映射，再由 `FileUploadZone` 正确传给 `react-dropzone`。
  - 保持 quick-mode 业务语义不变，只修 dev 启动依赖与上传过滤声明。
- Compatibility considerations:
  - 不 reopen `058`；本计划单独承接这次 frontend hygiene 修复。
  - 不修改 backend 上传接口或文件处理逻辑；只修 frontend accept 声明与本地 dev font dependency。
  - 不把这次修复与 `059` 的 canvas discovery 混合。
- Rollback plan:
  - 字体策略与上传 accept 映射都可独立回滚，不影响 runtime workspace 主流程。
