## 1. 当前问题
- 症状：
  - `build_runtime_view_for_task[_sync]()` 当前会在 read/projection 路径中调用 reconcile，并可能直接将 authoritative runtime 标记为 failed。
  - quick current-run selector 会吞掉 runtime projection integrity error，导致损坏状态被静默伪装成“没有当前 run”。
  - `FastAPI lifespan` 默认可启动 quick-runtime reconcile loop，使 API 进程成为 runtime mutation owner。
- 影响：
  - 违反 read-model purity、single reconcile owner、error transparency 三条核心边界。
  - 会把 runtime authoritative mutation 分散到高扇出入口，形成第二条控制面主线。
  - 已成为 `PLAN-20260329-038` Phase 4 的阻塞项。
- 发现来源：
  - 该问题由 `PLAN-20260329-038` 的 Phase 3 / Phase 4 预审发现；`038` 保留 blocker 留痕，但不拥有本专项实施。

## 2. 偏离分析
- 预期行为：
  - runtime/read-model builder 只做 projection，不做 authoritative mutation。
  - stale runtime reconcile 只由单一显式 control-plane maintenance 入口拥有。
  - current-run discovery 若遇到 runtime projection integrity error，应显式 surfaced，而不是静默跳过。
- 实际行为：
  - runtime view builder 内部直接 reconcile 并可能 fail session。
  - current-run selector 捕获 `ValueError` 后继续扫描候选任务。
  - API 进程默认通过 lifespan 启动 reconcile loop。
- 根因假设：
  - 在 keepalive / stale runtime 修复压力下，reconcile 被作为“读路径自愈”插入到最容易触达的高频入口，而不是收敛到单一 control-plane mutation surface。

## 3. 目标边界
- 本计划只处理 runtime boundary 根因修复，不承接 `038` 的内容质量、scene/script/image/video contract 主线。
- 本计划关闭以下问题：
  - read-path mutate-on-read
  - API process default reconcile owner
  - selector silent suppression of projection integrity errors
  - lease/heartbeat/reconcile owner 语义未冻结
- 本计划不做：
  - 泛化的 runtime 重构
  - queue/worker 调度体系重写
  - 视频内容质量逻辑调整

## 4. 实施方案
- Patch strategy：
  - 从 `build_runtime_view_for_task[_sync]()` 中移除 reconcile 调用，保留纯 projection 语义。
  - 收紧 current-run selector 和相关 API 对 projection integrity error 的处理，改为显式 surfaced。
  - 从 `FastAPI lifespan` 中移除 quick-runtime reconcile loop 的注册，而不是仅把默认配置翻转；保留脚本或单次 helper 作为显式 control-plane maintenance 入口。
  - 用文档和测试冻结 `lease_owner`、heartbeat sender、reconcile owner 三者的职责关系：`lease_owner` 表示 control-plane 对当前 attempt 授予 lease 的 owner / issuer identity；heartbeat sender 属于 execution host，只能凭 `lease_token` 续约现有 lease；reconcile owner 仅属于显式 maintenance 入口。
  - queued execution host 允许承载 heartbeat sender 上下文，但不得升级为 decision owner、reconcile owner、或新的 runtime mutation mainline。
- Compatibility considerations：
  - 显式 reconcile 脚本或单次 maintenance 入口保留，以免丢失必要治理能力。
  - `038` 的 blocker 关系必须留痕，不回写历史阶段内容。
  - 本计划不产出新的 owner matrix、runtime carrier、或跨模块治理抽象；语义冻结仅覆盖现有 lease issuance / heartbeat sender / reconcile owner 三条边界职责。
  - 本计划允许触达 orchestrator / queued execution host / config / queue tests，仅用于闭合上述 ownership handoff；不授权泛化 runtime / worker / queue 治理改造。
- Rollback plan：
  - 若修复引入 runtime discoverability 回归，仅回滚本计划触及的 runtime boundary 文件，不回滚 `038` Sprint 1-3 的内容 contract 成果。
